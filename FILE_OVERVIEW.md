## Top-Level Files

- **ARCHITECTURE.md**: Describes the system architecture, data flow, and main modules.
- **package.json**: Lists project dependencies and scripts for building/running the React app.
- **README.md**: Project introduction and usage instructions.

## Folders

- **build/**: Production build output (static files) generated by Create React App for deployment.
  - **index.html**: Main HTML file loaded in the browser.
  - **static/**: Bundled JS and CSS assets for the app.
- **public/**: Static files served directly (e.g., `index.html`, `manifest.json`, `robots.txt`).
- **src/**: Source code for the React application.

## Source Files (`src/`)

- **App.js**: Main React component. Manages state, handles user input (SQL text or file upload), parses SQL, generates Mermaid code, and provides export/share actions.
- **Diagram.js**: Receives Mermaid code and renders it as an SVG diagram using the Mermaid library. Adds pan and zoom interaction to the diagram.
- **App.css**: Styles for the main app UI.
- **index.css**: Global styles for the app.
- **index.js**: Entry point for the React app, renders `App` into the DOM.

## How the App Works

1. **User Input**: User pastes SQL or uploads a `.sql` file.
2. **Parsing**: `parseSQL` in `App.js` extracts tables and relationships from SQL.
3. **Transformation**: `convertToMermaid` in `App.js` converts the parsed model to Mermaid ER diagram syntax.
4. **Rendering**: `Diagram.js` renders the Mermaid code as an interactive SVG.
5. **Export/Share**: Users can export the diagram as PNG or share a link with the SQL encoded in the URL.
